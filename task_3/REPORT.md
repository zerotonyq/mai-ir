## Токенизация

### Правила

1. Токен - это последовательность латинских букв, цифр и кириллических букв (UTF‑8 диапазоны `D0/D1 xx`).
2. Внутри токена разрешены дефис `-` и апостроф `'`, если они стоят **между** двумя буквенно‑цифровыми символами (например, `e-mail`, `rock'n'roll`).
3. Регистр приводится к нижнему (ASCII + русские А‑Я/Ё в UTF‑8).
4. Если включён флаг `--strip-html`, текст между `<` и `>` игнорируется.

### Достоинства

- Простые и быстрые правила: линейная обработка, минимум ветвлений.
- Поддержка латиницы и кириллицы без внешних библиотек.
- Сохраняются распространённые составные слова с дефисом и апострофом.

### Недостатки

- Нет поддержки других алфавитов (греческий, китайский и т.д.).
- Числа с точкой/запятой дробятся (`3.14` → `3`, `14`).
- Аббревиатуры с точками дробятся (`U.S.A.` → `u`, `s`, `a`).
- Неразрывные и типографские дефисы (–, —, ‑) не считаются частью токена.
- Упрощённое «снятие» HTML может терять текст внутри атрибутов/скриптов.

### Примеры неудачных токенов и возможные исправления

- `3.14` → `3`, `14`. Исправление: разрешить «точку внутри числа», если по обе стороны цифры.
- `U.S.A.` → `u`, `s`, `a`. Исправление: разрешить последовательности `A.` в аббревиатурах.
- `C++` → `c`. Исправление: добавить правило для `+` внутри специальных слов (языки, C++, C#).
- `слово‑слово` (неразрывный дефис `U+2011`) разбивается. Исправление: добавить обработку типографских дефисов.
- `e‑mail` (типографский дефис) разбивается. Исправление: нормализовать все дефисы к `-`.

## Статистика токенизации

Команда для запуска:

```
./tokenize --list filelist.txt --out freq.csv --strip-html
```

**Результаты (последний запуск):**

- Объём входных данных: 96959569 байт (94687.079 КБ)
- Количество токенов: 7734851
- Средняя длина токена: 11.098 байт
- Уникальных терминов: 200506
- Время выполнения: 2.735073 секунд
- Скорость токенизации: 34619.579 КБ/с

### Зависимость времени от объёма

Алгоритм токенизации работает линейно по числу байт входа, т.е. ожидается **линейная** зависимость времени от объёма данных. Для демонстрации можно запустить программу на подмножествах корпуса и построить график `T(N)`; точки должны лежать близко к прямой.

### Оптимальность скорости и ускорение

Текущая скорость близка к оптимальной для однопоточного чтения, но можно ускорить:

- увеличить размер буфера чтения;
- использовать `mmap`/memory‑mapping (на Linux) или `CreateFileMapping` (Windows);
- распараллелить по файлам;
- применить SIMD‑классификацию ASCII‑байтов;
- ускорить хеш‑таблицу (меньше коллизий, better load factor).

## Закон Ципфа

Для построения графика используется `freq.csv` и скрипт `plot.py`:

```
python plot.py --input freq.csv --out zipf.png
```

На графике в логарифмических шкалах накладываются:

- эмпирическое распределение частот терминов;
- теоретическая кривая закона Ципфа `f(r) = C / r`, где `C = f(1)`.

### Причины расхождений с законом Ципфа

- Корпус конечен и тематически ограничен: «хвост» распределения короче и шумнее.
- Токенизация допускает артефакты (аббревиатуры, числа, HTML‑шум).
- Смешение источников с разными стилями (новости, заголовки, цитаты).
- Морфология (варианты словоформ) уменьшает «чистую» частоту лемм.

## Доп. задание: закон Мандельброта

При запуске с `--fit-mandelbrot` подбираются параметры `alpha`, `b`, `c` для модели:

```
f(r) = c / (r + b)^alpha
```

**Подобранные параметры (вставить после запуска):**

- `alpha = 1.180`
- `b = 0.000`
- `c = 16048780.278`

Обычно кривая Мандельброта лучше аппроксимирует верхнюю часть распределения, сглаживая отклонения для малых рангов.
В моём запуске аппроксимация вышла завышенной относительно эмпирики, поэтому на графике зелёная линия может быть выше синей.
