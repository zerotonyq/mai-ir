# Отчёт по корпусу документов (Kommersant + RBC)

## Источники данных
- Коммерсантъ (kommersant.ru): статьи из sitemap `sitemaps/index_sitemap.xml`.
- РБК (rbc.ru): статьи из `sitemap_index.xml`.

## Характеристики корпуса
Текст документов представляет собой HTML‑страницы статей. Внутри обычно есть:
- заголовок;
- автор;
- дата публикации;
- основной текст статьи (абзацы, подзаголовки);
- дополнительные элементы (изображения, теги, связанные материалы).

Текст для корпуса выделяется из HTML путём извлечения текста из тегов `p`, `h2`, `h3` внутри статьи.

## Поисковики для существующего корпуса
Можно использовать:
- поиск на сайтах Коммерсантъ и РБК;
- поиск Google с ограничением по домену через `site:kommersant.ru` и `site:rbc.ru`.

Примеры запросов:
- `site:kommersant.ru экономика прогноз`
- `site:kommersant.ru "наука"`
- `site:rbc.ru рынок акции`
- `site:rbc.ru "IT"`

Недостатки выдачи:
- в результатах встречаются не статьи (фото, видео, спецпроекты);
- выдача смешивает разные типы материалов;
- возможны дубликаты и редиректы.

## Как получить данные
1. Запустить скраппер и сформировать корпус.
2. Убедиться, что статистики заполнились автоматически.
3. Загрузить корпус в MongoDB.
